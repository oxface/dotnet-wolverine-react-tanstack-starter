# Architecture Decision Records

Use ADRs for technical-only decisions that do not need an OpenSpec behavior
spec. Good ADR candidates include dependency selection, architecture
boundaries, process or tooling changes, deployment posture, and accepted
tradeoffs.

Use OpenSpec changes instead when the decision changes user-visible behavior,
runtime behavior, workflow semantics, or requires implementation tasks across a
feature slice.

Create ADRs from `0000-template.md`, number them sequentially, and keep each one
short enough for human review.
